# ADOFAI 视效制作知识库

这组文档用于指导 AI 在谱师已经给出节奏点、楼层范围和段落意图后，为 ADOFAI 编辑器设计与实现视效。它不负责听歌、拆节奏、自动判断重拍，也不替代谱师的音乐决策。

## AI 的工作边界

开始设计前，至少需要得到：

- 目标关卡与当前 revision。
- 需要处理的 floor 范围。
- 由谱师标出的段落边界、强调点和转场点。
- 每段的视觉意图，例如冷静、压迫、轻盈、失真或高潮。
- 主色、辅色、禁用效果和闪光强度限制。
- 哪些现有事件或装饰必须保留。

信息不足时应明确指出缺失项，不要自行从音频推测节奏或情绪。

## 核心结论

成熟视效不是把所有效果同时开满，而是建立视觉层级：

```text
轨道与行星可读性
        ↓
镜头决定观看方式
        ↓
装饰建立空间与主题
        ↓
色彩和光效建立段落身份
        ↓
闪光、震动、失真只强调少量关键点
```

在研究的 96 个 Workshop 主谱中，`MoveCamera` 出现率为 100%，`MoveTrack` 为 97.9%，`Flash` 为 95.8%，`SetFilter` 为 89.6%。稳定的基础骨架是镜头、轨道和闪光；装饰、滤镜、Bloom 与震动用于形成差异和强调。

即使在视效活跃的 floor 上，65.5% 也只使用一种视觉事件类型，84.6% 不超过两种。好看的关键是选择和对比，不是事件数量。

## 通用制作流程

1. 读取当前编辑器状态和目标范围内现有事件。
2. 写出段落视觉说明：主体、空间、色板、运动方向、强度等级。
3. 先确定轨道可读性和相机基线。
4. 再建立背景、中景、前景装饰层。
5. 设计每个对象和临时效果的进入、保持、退出、复位。
6. 最后添加少量强调效果。
7. 分批修改、预览、检查遮挡和跳变，再继续下一段。
8. 全段通过后才保存关卡。

## 强制原则

- 永远优先保证轨道、行星和下一步路径可读。
- 每段只设一个主要视觉动作，其他效果为它服务。
- 临时效果必须有明确结束状态；不能把状态泄漏到下一段。
- 不复制 Workshop 中的具体美术资产或整段事件，只学习构图和运动方法。
- 参数统计是起始尺度，不是审美答案；必须在游戏内预览后调整。
- 不把不同滤镜的 intensity 当成统一强度。
- 不让短 Tween 随意重叠；游戏经常会先完成旧 Tween，导致跳变。
