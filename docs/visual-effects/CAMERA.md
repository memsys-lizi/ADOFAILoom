# 运镜

## 1. 运镜的职责

相机不是装饰效果。它负责：

- 决定玩家看哪里。
- 控制空间尺度和段落密度。
- 连接不同构图。
- 在强调点制造冲击与回弹。
- 让轨道运动看起来有共同方向。

先设计镜头，再让装饰和轨道配合镜头；不要最后随手加 `MoveCamera`。

## 2. relativeTo 的使用

| relativeTo | 适合用途 | 风险 |
|---|---|---|
| Player | 常规跟随、稳定读轨、推近拉远 | 过度偏移会让前方轨道离开视野 |
| Tile | 定点构图、展示特定场景或图形 | 不同 floor 的同一 position 不是同一世界位置 |
| Global | 固定世界构图 | 对路径变化非常敏感 |
| LastPosition | 增量位移和增量旋转、冲击与回弹 | 连续累加后必须知道最终状态 |
| LastPositionNoRotation | 保持当前位置基准但重设旋转 | 容易与上一镜头的累积状态混淆 |

常规段落优先 `Player`；构图镜头使用 `Tile`；只有明确知道前一镜头状态时才使用 `LastPosition`。

## 3. 从样本得到的尺度

以下数据来自 96 个主谱，只表示成熟谱中的常用起点：

- 26.9% 的镜头是 duration=0 的切镜。
- 30.8% 在 1 拍以内完成。
- 33.3% 使用 1–4 拍。
- 53.2% 的显式 position 为 `[0,0]`；大量镜头只改变 zoom 或 rotation。
- 显式 rotation 中，0° 占 47.3%；非零旋转主要集中在 5°、10°、15°等小角度。
- zoom 最常落在 100–180；超过 250 的镜头只占少数。

因此默认策略应是“小幅、清楚、有目的”，不是每次都同时移动、旋转和缩放。

## 4. 基础镜头类型

### 稳定跟随

- `relativeTo=Player`
- position 接近 `[0,0]`
- rotation 接近 `0`
- zoom 取本段基线

用于普通段落和强效果后的恢复。稳定镜头不是缺少设计，而是给其他动作留出阅读空间。

### 推近与拉远

- 推近：提高 zoom，强化密度与压迫。
- 拉远：降低 zoom，展示大范围轨道、装饰或空间变化。
- 常用 duration 为 1–4 拍，`OutCubic/OutQuad/OutSine` 用于快速建立新构图。
- 长段缓慢变化使用 `InOutSine` 或近似平滑 ease。

不要同时大幅改变 position、rotation 和 zoom，除非它们表达同一方向。

### 轻微倾斜

- 2–5°：持续氛围或轻微不安。
- 5–15°：明确的方向变化。
- 90°/180°：结构性翻转，应视为段落事件而不是普通强调。

倾斜后必须决定是保持新构图，还是在明确时间内回到 0°。不要留下无意的残余角度。

### 冲击与回弹

可靠结构：

1. duration=0 设置冲击状态，例如小幅位移、倾斜或 zoom 改变。
2. 用 1–2 拍 `OutCubic`、`OutBack` 或 `OutElastic` 回到基线。

Workshop 中常见同 floor 的两事件结构：先瞬时设置 ±10° 或新的 zoom，再用约 2 拍回到原值。只选一到两个通道做冲击，避免所有通道同时跳变。

### 定点展示

- 使用 `Tile` 把镜头固定到事件 floor 周围。
- 让 position 与装饰主体构图一致。
- 进入前确认玩家和后续轨道仍可读。
- 离开时显式恢复 `Player` 或下一个构图基准。

## 5. 镜头与其他系统的配合

### 镜头 + 轨道

这是样本中最稳定的组合。镜头给出总体运动方向，轨道用较小的相对运动形成景深。例如镜头向右推进时，轨道从左侧滑入，而不是随机向另一个方向飞行。

### 镜头 + 装饰

背景层运动慢、位移小；前景层运动快、位移大。镜头改变时，至少保留一个稳定层，否则整个画面会失去参照物。

### 镜头 + Flash

切镜可以由短 Flash 遮挡。Flash 应覆盖切换瞬间，不应长时间掩盖新构图。背景 Flash 更适合换色，前景 Flash 更适合冲击或遮挡硬切。

### 镜头 + Shake

Shake 是镜头上的局部噪声，不是第二套运镜。先确定基础镜头目标，再叠加短 Shake；不要让持续镜头 Tween 与高强度 Shake 长时间竞争。

## 6. 防跳变规则

游戏对同一相机通道的新 Tween 经常会先完成旧 Tween，再开始新 Tween。为避免跳变：

- 同一通道的相邻 Tween 不得无意重叠。
- 新事件只修改需要改变的通道，其他通道应禁用而不是重复写当前值。
- 长镜头中需要插入冲击时，先计算冲击后应恢复到哪个确定状态。
- `LastPosition` 的累积链必须可追踪；不能根据“看起来差不多”继续叠加。

## 7. 运镜自检

- 玩家能否在镜头运动全程看到当前与下一段轨道？
- 这个镜头的主要动作能否用一个动词描述？
- position、rotation、zoom 是否有无意义的重复赋值？
- 是否存在同通道 Tween 重叠？
- 强镜头后是否有稳定镜头建立对比？
- 进入 Tile/Global 后是否显式离开？
- 使用 LastPosition 时是否知道精确起点和最终状态？
