# 装饰与装饰动画

## 1. 把装饰当作有身份的对象

装饰不是“在某拍临时放一张图”。`AddDecoration` 在关卡加载时声明对象，真正的入场、保持和退场由 `MoveDecorations` 完成。

每个装饰在创建前应明确：

- 所属段落。
- 视觉角色：背景、中景、前景、粒子、文字、遮罩或转场。
- 初始状态。
- 入场方式。
- 保持期间是否运动。
- 退场和复用方式。

推荐 tag：

```text
p{段落}_{角色}_{名称或序号}
```

例如：

```text
p03_bg_city
p03_mid_ring_01
p03_fg_vignette
p03_particle_dust
p03_text_title
p03_transition_black
```

不要用一个超长 tag 列表管理整首谱，也不要依赖无 tag 的 `NO TAG` 集合。

## 2. 样本中的稳定做法

96 个主谱共包含 17,669 个 AddDecoration：

- 92% 以上带非空 tag。
- 8,959 个初始 opacity 为 0。
- 被动画控制的装饰中，初始 opacity=0 是最常见状态。
- 绝大多数 tag 只标识一个对象；少数 tag 用于控制一个小组。
- 常见 parallax 层为 0、25、50、80、99/100 附近。

这说明成熟谱通常先把对象注册在确定状态，再通过 tag 驱动，而不是在事件发生时随意寻找图片。

## 3. 三层装饰结构

### 背景层

- 正 depth。
- 低 parallax 或固定层。
- 低对比度、低细节、慢速运动。
- 不能和轨道争夺边缘与亮度。

适合远景、雾、渐变、星空、大型纹理和缓慢几何图形。

### 中景层

- 中等 parallax。
- 承担主题 motif、建筑、角色轮廓、主图形。
- 运动方向与镜头或段落方向一致。

### 前景层

- 负 depth。
- 常用 Camera/CameraAspect 固定在画面空间。
- 只用于边框、暗角、遮罩、文字、冲击图形和转场。
- 长时间前景遮挡必须低透明度并避开轨道核心区域。

## 4. 坐标与放置方式

- Tile/Global/Planet 的 position 使用 tile 单位。
- Camera/CameraAspect 使用屏幕相对坐标，不按 tileSize 解释。
- `MoveDecorations.positionOffset` 即使目标是 Camera 装饰，运行时仍按 tileSize 处理；不要假设它与 AddDecoration 的 Camera position 完全对称。
- `rotationOffset` 是绝对目标角度，不是自然累加。
- scale、opacity、parallax 是绝对目标百分比。
- 多 tag 查询是并集，不是同时满足全部 tag。

## 5. 装饰入场

### 淡入

1. AddDecoration 初始 opacity=0。
2. MoveDecorations 在 1–4 拍内到 100。
3. 使用 OutSine/OutCubic。

适合背景、雾、文字和需要平稳出现的层。

### 缩放入场

1. 初始 scale 接近 0 或明显小于目标。
2. 1–4 拍到目标 scale。
3. OutCubic 适合稳定图形，OutBack 适合带设计感的弹入。

不要对写实背景使用强 Bounce/Elastic。

### 位移入场

1. 初始位置在构图外或目标附近。
2. 用 1–4 拍进入。
3. 运动方向应与镜头、轨道或构图线一致。

只有确实要改的轴才传值；保留轴应使用该事件类型的禁用/null 语义，不能补成 0。

### 遮切入场

前景黑幕、白幕或形状先覆盖画面，在遮挡期间完成背景、相机和装饰组切换，再退场。遮挡对象必须有明确透明终点。

## 6. 保持动画

保持动画应比入场慢，并且不抢主体：

- 背景缓慢漂移。
- 中景轻微缩放或旋转。
- 粒子使用方向一致的小范围移动。
- 文字通常保持稳定，只在进入和离开时动画。
- 对称图形可以使用反向旋转建立层次。

需要连续累积 position 时才使用 LastPosition。普通 relativeTo 的 positionOffset 会从基础 startPos 计算，不会自动接着当前位置累积。

## 7. 退场与清场

可靠退场：

- opacity 到 0。
- scale 缩小。
- 沿既有运动方向移出画面。
- 1–4 拍，InCubic/InBack。

完成后保持隐藏，直到明确复用。段落切换按角色 tag 清场，例如先清 `p03_fg_*` 和 `p03_transition_*`，再决定哪些背景层延续到下一段。

## 8. Parallax 设计

Parallax 的目标是制造空间差，不是随机填数：

- 0 附近：远景或固定世界层。
- 25 附近：远景。
- 50 附近：中远景。
- 80 附近：近景。
- 99/100 附近：贴近轨道或相机运动。

同一段至少保留两个明显不同的 parallax 层。所有装饰都用 100 会让场景像贴纸；所有装饰都用 0 又会缺乏空间运动。

## 9. 图形 motif

优质谱经常用少量可识别的图形贯穿全曲：圆环、十字、网格、字形、行星轨迹、线条或某种材质。做法是：

- 保持形状身份。
- 在不同段落改变色彩、尺度、层级和运动方式。
- 高潮组合多个既有 motif，而不是突然引入完全无关的素材。

一个稳定 motif 比几十种随机装饰更容易形成作品辨识度。

## 10. 装饰自检

- 每个 tag 是否有清楚的段落与角色含义？
- 初始状态是否与首次可见状态一致？
- 是否存在永远不退场的临时前景？
- parallax 是否形成至少两个空间层？
- 前景是否遮挡轨道或行星？
- rotationOffset 是否按绝对目标计算？
- visible、depth、换图和遮罩是否被误认为可随 duration 渐变？
- color alpha 与 opacity 相乘后是否仍是预期透明度？
